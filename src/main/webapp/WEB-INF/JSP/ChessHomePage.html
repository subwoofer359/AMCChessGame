Chess Game

Introduction

This chess game has been developed using the MVC (model-view-controller) design pattern. The chess game model is located on a server. The view in the game uses HTML5, SVG, CSS and JavaScript to render the chessboard to the player. The controller uses messages sent from the player over a websocket connection to make moves in the chess game. The updated view is then sent to both players over the websocket connection. The chess game model is written in Java and is stored on an Apache Tomcat server. The server code uses the Spring framework to create http request and websocket(STOMP) controllers. On the client side, the stomp.js library is used to send websocket(STOMP) message to the server and to receive push updates. The interact.js library is used to allow chess pieces to be moved on the chessboard view by the player.
